# TOR Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add on-demand TOR routing for all outbound scan traffic, activated from the Proxy Settings page, using a dedicated Docker container that starts/stops when the toggle is flipped.

**Architecture:** A `r3ngine-tor` Docker container (custom Debian image with `tor`) starts when `use_tor=True` is saved and stops when it's disabled. The existing `ProxychainsWrapper` is patched to emit `socks5 tor 9050` as the proxy when TOR mode is active, so all proxychains-wrapped tools automatically route through TOR. DNS leaks are blocked via the `proxy_dns` directive already present in `write_temp_config`. A `TorNewCircuitActivity` fires at the start of every master scan and subscan to rotate exit nodes.

**Tech Stack:** Python `stem` library (TOR control protocol), Python `docker` SDK (already in requirements), Django ModelForm extension, Temporal activity, React/MUI frontend.

- [ ] **Step 3: Create `web/reNgine/tor_manager.py`**

```python
import socket
import time
import logging

import docker
from django.conf import settings

logger = logging.getLogger(__name__)

TOR_CONTAINER_NAME = 'r3ngine-tor'
TOR_SOCKS_HOST = 'tor'
TOR_SOCKS_PORT = 9050
TOR_CONTROL_HOST = 'tor'
TOR_CONTROL_PORT = 9051


class TorUnavailableError(Exception):
    pass


class TorStartError(Exception):
    pass


class TorManager:

    def _get_client(self):
        try:
            return docker.from_env()
        except docker.errors.DockerException as e:
            raise TorUnavailableError(f"Docker socket not available: {e}")

    def start(self):
        client = self._get_client()
        try:
            container = client.containers.get(TOR_CONTAINER_NAME)
        except docker.errors.NotFound:
            raise TorStartError(
                f"Container '{TOR_CONTAINER_NAME}' not found. "
                "Ensure the tor service is defined in docker-compose.yml."
            )
        if container.status != 'running':
            container.start()
        self._wait_for_ready()

    def stop(self):
        try:
            client = self._get_client()
            container = client.containers.get(TOR_CONTAINER_NAME)
            if container.status == 'running':
                container.stop(timeout=10)
        except (docker.errors.NotFound, TorUnavailableError):
            pass

    def is_running(self):
        try:
            client = self._get_client()
            container = client.containers.get(TOR_CONTAINER_NAME)
            return container.status == 'running'
        except (docker.errors.NotFound, TorUnavailableError):
            return False

    def _wait_for_ready(self, timeout=30):
        deadline = time.time() + timeout
        while time.time() < deadline:
            try:
                sock = socket.create_connection((TOR_SOCKS_HOST, TOR_SOCKS_PORT), timeout=2)
                sock.close()
                return
            except OSError:
                time.sleep(1)
        raise TorStartError(
            f"TOR SOCKS5 on {TOR_SOCKS_HOST}:{TOR_SOCKS_PORT} "
            f"did not become ready within {timeout}s"
        )

    def new_circuit(self):
        password = getattr(settings, 'TOR_CONTROL_PASSWORD', '')
        try:
            from stem import Signal
            from stem.control import Controller
            with Controller.from_port(
                address=TOR_CONTROL_HOST, port=TOR_CONTROL_PORT
            ) as ctrl:
                ctrl.authenticate(password=password)
                ctrl.signal(Signal.NEWNYM)
                time.sleep(2)
        except Exception as e:
            logger.warning(f"[TorManager] Failed to rotate circuit: {e}")
            raise
```

@activity.defn(name="TorNewCircuitActivity")
def run_tor_new_circuit_activity() -> None:
    """Request a new TOR circuit (NEWNYM) to rotate the exit node.

    No-op when TOR mode is not active. Logs a warning on failure
    but never raises — a missed rotation is non-fatal.

    from scanEngine.models import Proxy
    from reNgine.tor_manager import TorManager
    proxy = Proxy.objects.first()
    if not proxy or not proxy.use_tor:
        return
    try:
        TorManager().new_circuit()
        activity.logger.info("[TorNewCircuitActivity] New TOR circuit requested successfully")
    except Exception as e:
        activity.logger.warning(f"[TorNewCircuitActivity] Circuit rotation failed (scan continues): {e}")
